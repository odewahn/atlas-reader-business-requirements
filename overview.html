Overview

This document contains the business requirements documents (BRDs) for the "Atlas Reader" project.  The goal of the document is to provide a central point for collecting and discussing features for the reader and the related initiatives, such as full content search .

 

 

To write a new BRD for a feature, create a new file named after the feature (e.g. better-analytics.html), and copy/paste the contents of COPYME.html into your new document (yes, we need file templates). When you're done writing, put the filename underneath the feature description in the backlog.

Before every product council meeting, the council leader will look through the parking lot for new BRD's, and we will discuss the priority of these features. Instead of going through the entire parking lot every time, we will only discuss features with new BRD's, and how they relate to the existing prioritized features.
