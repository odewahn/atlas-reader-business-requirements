Feature Description

SOLR is a search indexer that will crawl the Content Storage unit and provide a unified API that can be used to retrieve indexed content.

  • Build a minimal reader we need for our project to go live in August.
  • Build the most minimal Reader possible to meet the needs of the project.  
  • Provide the API’s for the Content Guardian that controls access to content.
  • Drive the URL structure for content for Google to crawl.
  • Consume HTML output from Atlas.  (?)  We won’t be directly integrated into the Atlas data structures to pull our book content.
  • Provide a loosely coupled API driven set of services that you can use in the future.
